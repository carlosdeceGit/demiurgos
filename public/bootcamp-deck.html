FUNCIONALIDADES · PARTE 1

¿Qué puedes hacer?

  • Chat con el Director Creativo — Conversación contextualizada con tu perfil e historial.
  • Generación de propuestas — Ideas + por qué ahora + guion, listas para publicar.
  • Conexión y subida de archivos — Google Drive, PDFs, notas propias como fuente de inspiración.

FUNCIONALIDADES · PARTE 2

  • Tendencias en tiempo real — Lo que está pasando en tu sector, integrado como contexto.
  • Ajustes de coste — Control granular sobre qué modelos y cuánto gasto en cada sesión.

EL PROCESO

Cómo se construyó.

1

Construir el prompt

Primero se levantó un GPT para diseñar un prompt sólido que definiera el comportamiento del director creativo.

2

Levantar el proyecto

Con el prompt como base, se inició la arquitectura completa con Claude Code: Next.js, Supabase, autenticación, API.

3

Iterar (y pelearse con la IA)

Debugging constante, decisiones en caliente y aprendizaje forzado de todo lo que hacía falta.

LA PARTE QUE NADIE CUENTA

La pelea con la IA.

🔧 Bloqueos técnicos

Merges rotos, pull requests conflictivos, variables de entorno que desaparecen, builds que fallan sin razón.

🤯 Decisiones en caliente

¿Cambio de arquitectura a mitad? ¿Descarto la feature? ¿Uso otro modelo? Con el reloj corriendo.

💸 Control de costes

Cada prompt cuesta. Aprender cuándo es eficiente y cuándo estás quemando dinero en iteraciones inútiles.

🧠 Contexto que se pierde

La IA no siempre recuerda. Tú sí tienes que recordar por los dos y escribir prompts más precisos.
